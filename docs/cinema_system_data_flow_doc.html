Cấu trúc AuditLog

  • action: tên hành động (approve_partner, ban_user...)
  • description: mô tả chi tiết
  • category: nhóm (user, partner, movie, system...)
  • severity: low / medium / high / critical
  • actorId/actorEmail: ai thực hiện
  • targetType/targetId: đối tượng bị tác động
  • ip, device, location: context thiết bị
  • meta: JSON extra data

Các action được audit

  • Duyệt/từ chối Partner request
  • Duyệt/từ chối Movie
  • Ban/unban User/Partner
  • Duyệt Withdrawal requests
  • Thay đổi SystemSetting
  • Bật/tắt FeatureFlag
  • Xử lý Report content
  • Broadcast notification gửi

Background jobs, cleanup tasks và các tác vụ tự động

Job: Expire Order & Release Seat

  • Scan Order WHERE status=PENDING AND expiresAt < now
  • Order.status → EXPIRED
  • Seat.status → AVAILABLE (giải phóng lock)
  • Ticket nếu có → status=CANCELLED
  • Chạy mỗi 1-2 phút
OrderSeatTicket

Job: Showtime Status Update

  • Showtime.startTime <= now → status=STARTED
  • Showtime.endTime <= now → status=ENDED
  • Ticket CONFIRMED nhưng showtime ENDED → PASSED
  • Cập nhật bookedSeats / availableSeats
  • Chạy mỗi 5 phút
ShowtimeTicket

Job: Cleanup Tokens

  • Delete PasswordResetToken WHERE expiresAt < now
  • Delete EmailVerificationToken WHERE expiresAt < now
  • Delete Session WHERE isActive=false AND expiresAt < now
  • Chạy hằng ngày (cron daily)
PasswordResetTokenEmailVerificationTokenSession

Job: Daily Partner Revenue Report

  • Tính doanh thu ngày cho mỗi Partner
  • Tạo Notification: PARTNER_DAILY_REVENUE
  • Cập nhật PartnerWallet.totalEarned
  • Chạy cuối ngày (23:59)
TransactionPartnerWalletNotification

Job: Email Queue Worker

  • Scan ScheduledEmailNotification WHERE status=PENDING
  • scheduledFor <= now → send email
  • status=SENT với sentAt, hoặc FAILED + failedReason
  • Retry FAILED tối đa N lần
